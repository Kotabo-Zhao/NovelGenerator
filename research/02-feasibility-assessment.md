## 1. 功能可实现性矩阵

### 1.1 核心流程

| 功能 | 可行性 | 实现难度 | 关键技术 | 备注 |
|:---|:---:|:---:|:---|:---|
| **创意输入 → 世界观生成** | ✅ | ⭐ | LLM Prompt 工程 | 用 DeepSeek 一步生成结构化世界观 |
| **角色自动创建** | ✅ | ⭐⭐ | LLM + 模板 | 输入 "修仙+废柴逆袭" → 自动生成 5-10 个角色卡 |
| **大纲/细纲生成** | ✅ | ⭐⭐ | LLM + 分层规划 | 卷纲 → 章纲 → 场景级大纲 |
| **逐章正文写作** | ✅ | ⭐⭐⭐ | LLM + 上下文管理 | 核心难点是上下文裁剪，见下文 |
| **角色一致性保持** | ⚠️ | ⭐⭐⭐⭐ | 结构化角色卡 + 每章注入 | 关键：每次写作前注入当前相关角色状态 |
| **伏笔追踪/回收** | ⚠️ | ⭐⭐⭐⭐ | 伏笔清单 + LLM 审计 | 需要专门 Agent 来检查"埋了哪些坑没填" |
| **长期记忆** | ⚠️ | ⭐⭐⭐⭐ | 分层摘要 + 语义检索 | 百万字场景下不可能全塞上下文 |
| **质量审计** | ✅ | ⭐⭐⭐ | LLM 多维度评分 | 角色一致性/情节逻辑/文风/节奏 |
| **自动修订修复** | ⚠️ | ⭐⭐⭐⭐ | 审计→定位问题→重写段落 | 需要精确定位"哪段出了问题" |
| **去 AI 味** | ✅ | ⭐⭐ | 规则检测 + LLM 润色 | 24 种模式：句式重复/词汇疲劳/过渡词泛滥 |
| **文风学习/仿写** | ✅ | ⭐⭐⭐ | RAG + 统计指纹 | 提取样本的句长/标点/词汇频率 |
| **断点续传** | ✅ | ⭐ | 状态文件 + 快照恢复 | JSON 状态保存，简单可靠 |
| **多模型路由** | ✅ | ⭐⭐ | API 网关 | 写作用 DeepSeek，审计用 GPT-4o-mini |
| **封面生成** | ✅ | ⭐⭐ | DALL-E / GPT-Image-2 | 可选 |
| **导出 PDF/EPUB** | ✅ | ⭐⭐ | 前端 JS 库 / 后端转换 | HTML→PDF 已有成熟方案 |

> **✅ 可直接实现 | ⚠️ 需要精心设计 | ❌ 当前不可行**

### 1.2 核心难点深度分析

**🔴 最大挑战: 长篇一致性**

问题：写到第 50 章，AI "忘记"了第 3 章埋的伏笔、第 20 章新增的角色关系。

解决方案（分层上下文策略）:

```
每次写作时的上下文组装:
┌─────────────────────────────────────┐
│ 第1层: 核心设定 (永远注入)           │ ← 500 tokens
│  世界观核心规则 / 主角档案 / 主线    │
├─────────────────────────────────────┤
│ 第2层: 近期上下文 (最近3章原文)      │ ← 6,000 tokens
│  上一章发生了什么 / 谁出场了         │
├─────────────────────────────────────┤
│ 第3层: 相关伏笔 (语义检索)           │ ← 1,000 tokens
│  从伏笔清单中找到当前应回收的        │
├─────────────────────────────────────┤
│ 第4层: 角色状态快照                   │ ← 1,500 tokens
│  本章将出场角色的当前状态/位置/关系  │
└─────────────────────────────────────┘
总计 ~9,000 tokens ← DeepSeek 128K 上下文完全够用
```

这套方案在 InkOS 和 oh-story 中都被验证有效，不是理论方案。

**🟡 次要挑战: 生成质量和节奏**

问题：AI 写的章节容易"流水账"——每章都差不多，缺乏高潮和节奏变化。

解决方案：结构化大纲驱动

```
每章大纲模板:
{
  "章节号": 23,
  "核心事件": "主角发现反派的真实身份",
  "情绪曲线": "紧张(开端) → 震惊(中段) → 愤怒(结尾)",
  "出场角色": ["主角", "反派", "配角A"],
  "伏笔操作": ["回收: 第5章的神秘信件", "埋设: 反派的下一步计划"],
  "本章钩子": "反派留下一句'你以为这就是全部了吗？'"
}
```

AI 严格按大纲写 → 大纲控制节奏 → 人工可随时调整大纲

## 2. 纯 HTML 前端方案分析

### 2.1 可行！但需要后端代理

```
你 (浏览器)                 后端 API 代理             LLM API
    │                           │                      │
    │ POST /api/generate        │                      │
    │ {prompt: "写第3章..."}    │                      │
    │ ─────────────────────────→│                      │
    │                           │ DeepSeek API call     │
    │                           │ ─────────────────────→│
    │                           │     streaming response│
    │                           │ ←─────────────────────│
    │   streaming SSE           │                      │
    │ ←─────────────────────────│                      │
    │                           │                      │
```

**为什么不直接从前端调 LLM API？**
- ❌ API Key 暴露在浏览器 → 任何人 F12 就能看到
- ❌ CORS 限制 → 大部分 LLM API 不允许浏览器跨域
- ❌ iOS Safari 的 PWA 限制更多

**解决方案: 极简后端**
- 一个几十行的 Python FastAPI / Node.js Express
- 只做三件事: 接收请求 → 转发 LLM → 流式返回
- 部署到 CloudStudio / Vercel /  Railway（免费额度够用）

### 2.2 前端技术选型

| 方案 | 优点 | 缺点 | 推荐度 |
|:---|:---|:---|:---|
| **纯 HTML+JS+CSS** | 零依赖、加载快、PWA 友好 | 复杂 UI 难维护 | ⭐⭐⭐ (MVP) |
| **React SPA** | 组件化、生态丰富 | 打包体积大、PWA 配置多 | ⭐⭐⭐⭐ |
| **Vue SPA** | 简单上手、中文社区好 | 同上 | ⭐⭐⭐⭐ |
| **Svelte** | 体积最小、性能好 | 生态较小 | ⭐⭐⭐ |
| **Alpine.js + Tailwind** | 极简、类 HTML 写法 | 大型应用难维护 | ⭐⭐⭐⭐ (推荐 MVP) |

**推荐: Vue 3 + Vite + Tailwind CSS**
- 体积小 (gzip ~40KB)
- 单文件组件，开发效率高
- 中文文档完善
- PWA 插件成熟 (vite-plugin-pwa)

### 3.2 首选方案: PWA (Progressive Web App)

**工作原理**:
```
https://novel.yourdomain.com
         ↓
   Safari 打开 → 点击「分享」→ 「添加到主屏幕」
         ↓
   主屏幕出现 App 图标 → 点击打开 → 全屏运行 (无地址栏)
         ↓
   看起来和原生 App 一模一样
```

**iOS PWA 支持情况 (iOS 16.4+)**:
- ✅ 添加到主屏幕 (独立图标)
- ✅ 全屏运行 (无 Safari UI)
- ✅ 推送通知 (iOS 16.4+ 支持 Web Push)
- ✅ 离线缓存 (Service Worker)
- ✅ 本地存储 (IndexedDB 可用于保存小说草稿)
- ⚠️ 后台同步有限 (切出 App 后 Service Worker 可能被挂起)

**PWA 实现要点**:

```json
// manifest.json
{
  "name": "NovelGenerator",
  "short_name": "小说生成",
  "start_url": "/",
  "display": "standalone",     // 全屏，像原生 App
  "background_color": "#0f1923",
  "theme_color": "#ff6b35",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192" },
    { "src": "/icon-512.png", "sizes": "512x512" }  // iOS 需要 512
  ]
}
```

```
// service-worker.js — 离线缓存
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open('novel-v1').then(cache => 
      cache.addAll(['/', '/app.js', '/app.css'])
    )
  );
});
```

### 3.3 次选方案: 微信小程序

**优势**:
- 用户体验最好 (微信原生体验)
- 微信登录 → 无需注册
- 云开发 → 自带后端/数据库/存储
- 推送通知 → 章节生成完成通知
- 支付 → 未来可以变现

**劣势**:
- 需要微信审核 (比 App Store 容易很多，通常 1-3 天)
- 小程序包体积限制 (2MB 主包 + 20MB 分包，够用)
- 不能用 npm 包中某些 Node.js API

**架构**:
```
微信小程序 (前端)
    ↓ wx.cloud.callFunction()
云函数 (后端) → DeepSeek API
    ↓
云数据库 (存储小说数据)
```

### 3.4 推荐: PWA + 微信小程序双端

| 离线写作 | PWA (添加到桌面) |
| 分享/演示 | PWA URL (任何人都能打开) |
| 开发测试 | PWA (无需审核) |

### 4.3 API 流式输出 (SSE)

```python
# backend/api.py
@app.post("/api/novel/generate")
async def generate_chapter(request: GenerateRequest):
    """流式生成章节，前端实时显示打字效果"""
    
    # 1. 组装上下文
    context = memory_manager.build_context(
        novel_id=request.novel_id,
        chapter_num=request.chapter_num
    )
    
    # 2. 构建 Prompt
    prompt = build_writer_prompt(context, request.chapter_outline)
    
    # 3. 流式调用 DeepSeek
    async def stream_response():
        async for chunk in deepseek_client.stream(prompt):
            yield f"data: {json.dumps({'text': chunk})}\n\n"
        yield f"data: {json.dumps({'done': True})}\n\n"
    
    return StreamingResponse(stream_response(), media_type="text/event-stream")
```

```javascript
// web/app.js — 前端流式接收
async function generateChapter(novelId, chapterNum) {
    const response = await fetch('/api/novel/generate', {
        method: 'POST',
        body: JSON.stringify({ novel_id: novelId, chapter_num: chapterNum }),
        headers: { 'Content-Type': 'application/json' }
    });
    
    const reader = response.body.getReader();
    const decoder = new TextDecoder();
    let content = '';
    
    while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        const text = decoder.decode(value);
        // 解析 SSE 格式
        for (const line of text.split('\n')) {
            if (line.startsWith('data: ')) {
                const data = JSON.parse(line.slice(6));
                if (data.done) return content;
                content += data.text;
                // 实时更新 UI
                updatePreview(content);
            }
        }
    }
}
```

## 总结

**可以做的**:
- ✅ HTML 网页 → PWA 添加到 iOS 主屏幕，体验 ≈ 原生 App
- ✅ 用户输入题材+灵感 → 自动生成世界观+角色+大纲+逐章正文
- ✅ 流式输出 → 实时看到 AI 写作过程，像打字机一样
- ✅ 每章可人工编辑/重生成 → 用户始终掌控

**需要注意的**:
- ⚠️ 需要后端代理 API Key (不能直接在前端调 LLM)
- ⚠️ 长篇一致性需要分层上下文策略 (已有多项目验证可行)
- ⚠️ CloudStudio 免费额度有限，长期使用建议 Railway 或自建

**一句话建议**:

> 用 **Vue 3 + FastAPI + DeepSeek** 做一个 **PWA**，部署到 **CloudStudio**，3 天出 MVP，用户打开网页就能用，iOS 添加到主屏幕就是"App"——这是最快的路径。
